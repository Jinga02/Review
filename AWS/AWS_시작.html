AWS (Amazon Web Services)란?

  • AWS는 아마존닷컴(Amazon.com)이 제공하는 클라우드 컴퓨팅 서비스로, 인터넷을 통해 가상 컴퓨팅 리소스를 제공한다.
  • AWS는 전 세계적으로 데이터 센터를 보유하고 있으며, 이를 통해 기업 및 개인 사용자가 인프라를 소유하지 않고도 컴퓨팅 리소스를 대여하고 사용할 수 있다.

  • On-premise

    • 기업의 서버를 클라우드 같은 원격 환경에서 운영하는 방식이 아닌, 자체적으로 보유한 전산실 서버에 직접 설치해 운영하는 방식이다.

  • Cloud Computing

    • 인터넷을 통해 컴퓨팅 리소스를 제공하는 기술로, 비용 효율성과 유연성이 높은 장점을 가지고 있다.
    • 필요한 만큼 리소스를 대여하여 사용할 수 있으며, 다양한 서비스를 제공한다.
    • 안정성과 보안성이 높아지며, 비즈니스를 더욱 효율적으로 운영할 수 있다.
    • AWS, Microsoft Azure, Google Cloud Platform 등 다양한 클라우드 서비스 제공업체에서 제공한다.

AWS의 특징

  1. 탄력적인 크기 조정
    • AWS는 사용자가 필요한 만큼 컴퓨팅 리소스를 추가하거나 제거할 수 있도록 탄력적인 크기 조정 기능을 제공한.
    • 이를 통해 사용자는 자신의 비즈니스에 맞는 컴퓨팅 리소스를 쉽게 조절할 수 있다.
  2. 다양한 서비스 제공
    • AWS는 EC2, S3, RDS, Lambda, DynamoDB, Route 53 등 다양한 서비스를 제공한다.
    • 이를 사용하여 웹 사이트, 애플리케이션, 데이터베이스, 스토리지, 인프라 등을 구축할 수 있다.
  3. 보안
    • AWS는 고급 보안 기술을 사용하여 사용자의 데이터와 컴퓨팅 리소스를 안전하게 보호한다.
    • AWS는 데이터 센터의 물리적 보안, 네트워크 보안, 데이터 보안, 인증 및 액세스 제어, 감사 추적 등 다양한 보안 기능을 제공한다.
  4. 비용 효율성
    • AWS는 필요한 컴퓨팅 리소스만큼 사용자가 지불하도록 하여 비용 효율적으로 서비스를 이용할 수 있도록 한다.
    • 또한, AWS는 예약 인스턴스, 스팟 인스턴스, 리조합 등 다양한 요금 제도를 제공하여 사용자가 더욱 비용을 절감할 수 있도록 한다.

AWS를 사용하는 이유

  1. 유연성
    • AWS는 다양한 서비스와 탄력적인 크기 조정 기능을 제공하여 사용자의 비즈니스에 맞는 컴퓨팅 리소스를 쉽게 구성할 수 있다.
    • 또한, AWS는 다양한 운영 체제 및 프로그래밍 언어를 지원하여 사용자가 더욱 유연하게 서비스를 이용할 수 있도록 한다.
  2. 안정성
    • AWS는 전 세계적으로 데이터 센터를 보유하고 있으며, 고급 보안 기술을 사용하여 사용자의 데이터와 컴퓨팅 리소스를 안전하게 보호한다.
    • 또한, AWS는 다양한 백업 및 복구 기능을 제공하여 데이터의 안정성을 보장한다.
  3. 확장성
    • AWS는 사용자의 비즈니스가 성장함에 따라 컴퓨팅 리소스를 확장할 수 있도록 설계되어 있다.
    • 이를 통해 사용자는 빠르게 성장하는 비즈니스에 대응할 수 있으며, 추가 비용 없이 서비스를 확장할 수 있다.
  4. 비용 효율성
    • AWS는 필요한 컴퓨팅 리소스만큼 지불하도록 하여 비용을 효율적으로 관리할 수 있도록 한다.
    • 또한, AWS는 요금 계산 및 예산 설정 기능을 제공하여 사용자가 더욱 효율적으로 비용을 관리할 수 있도록 한다.
  5. 전 세계적인 인프라
    • 또한, AWS는 요금 계산 및 예산 설정 기능을 제공하여 사용자가 더욱 효율적으로 비용을 관리할 수 있도록 한다.
    • 이를 통해 사용자는 지연 시간을 최소화하고 안정적인 서비스를 이용할 수 있다.

결론적으로, AWS는 사용자의 비즈니스에 맞는 컴퓨팅 리소스를 유연하게 구성할 수 있고, 안정적이고 안전한 서비스를 제공하며, 비용을 효율적으로 관리할 수 있는 클라우드 컴퓨팅 서비스이다.
따라서, 많은 기업과 조직들이 AWS를 사용하여 비즈니스를 운영하고 있다.

AWS VPC(Virtual Private Cloud)

  • 가상 네트워크 서비스로 퍼블릭 네트워크와 프라이빗 네트워크를 분리하고 모니터링할 수 있도록 해주는 서비스이다.
    • 퍼블릭 네트워크
      • 인터넷과 같은 공개 네트워크를 의미한다.
      • 퍼블릭 네트워크를 사용하면 인터넷을 통해 전 세계적으로 다양한 기기와 연결할 수 있다.
    • 퍼블릭 네트워크
      • 인터넷과는 분리된 네트워크를 의미한다.
      • 네트워크를 사용하면 내부망에서만 기기들이 서로 통신할 수 있다.
  • 네트워크 구성과 관련된 사실상 모든 기능을 담당하며, 자체 데이터센터에서 운영하는 기존 네트워크와 매우 유사한 형태이다.

AWS API Gateway

  • 어떤 규모에서든 개발자가 API를 손쉽게 생성, 게시, 유지 관리, 모니터링 및 보안 유지할 수 있또록 하는 완전관리형 서비스이다.
  • 서버의 "대문"과 같은 역할을 한다. 외부 사용자가 내부 서비스로 접근하는 가장 첫 단계
  • 사용자 인증과 권한 부여를 제공하는 인증 기능, API 호출의 속도 및 성능을 높이는 캐싱 기능, 다양한 백엔드 서비스와의 통합을 지원하는 통합 기능 등이 있다.
  • API Gateway를 사용하면, API의 생성과 관리가 쉬워지며, 안정적인 API 서비스를 빠르게 제공할 수 있다.

AWS ELB

  • 다수의 컴퓨팅 리소스(EC2, Lambda 등)를 부하 분산시켜주는 아주 중요한 서비스이다.
  • ELB를 사용하면 다수의 EC2 인스턴스를 로드 밸런서 앞에 놓고, 트래픽을 분산시키고, 더욱 안정적인 서비스를 제공할 수 있다.
  • 총 3가지 종류의 Balancer(Application Load Balancer, Gateway Load Balancer, Network Load Balancer)가 있으며, Application Load Balancer이 가장 많이 사용된다.

AWS S3 (Simple Storage Service)

  • 가장 오래된 AWS 서비스 중 하나로, 엄청난 내구성과 가용성을 자랑하며 이미지, 동영상, 오디오 파일과 같은 정적 파일들을 용이하게 관리하도록 돕는 스토리지 서비스이다.
  • 단순 파일의 저장 공간을 넘어서, 데이터를 저장하는 스토리지 플랫폼으로 발전했다.
  • 소규모부터 대규모의 데이터를 저장하고 관리하는데 사용되며, 웹 사이트, 모바일 애플리케이션, 백업 및 복원, 아카이브, 빅 데이터 분석, 기업 애플리케이션 등 다양한 분야에서 활용된다.

AWS CloudFront

  • AWS CloudFront는 Amazon Web Services의 글로벌 콘텐츠 전송 네트워크(CDN) 서비스이다. CDN은 사용자에게 웹 콘텐츠와 애플리케이션을 빠르게 전달하기 위해 전 세계의 데이터 센터와 프록시 서버로 구성되어 있다.
  • CloudFront는 사용자에게 가장 가까운 엣지 로케이션에서 콘텐츠를 제공함으로써 지연 시간을 최소화하고, 높은 전송 속도와 낮은 지연 시간을 보장한다. 이를 통해 웹 사이트, 애플리케이션 및 API의 성능을 향상시킬 수 있다.
  • CloudFront는 정적 콘텐츠(이미지, 스타일시트, JavaScript 파일 등)와 동적 콘텐츠(실시간 스트리밍, 대화형 애플리케이션 등) 모두 지원하며, 원본 서버로부터 콘텐츠를 가져와 엣지 로케이션에 캐싱한다.
  • AWS 서비스와의 통합이 용이하여, CloudFront는 Amazon S3, Amazon EC2, Elastic Load Balancing, AWS Lambda@Edge 등과 같은 다양한 AWS 리소스와 함께 사용할 수 있다.
  • 보안 측면에서, CloudFront는 SSL/TLS를 사용한 데이터 전송 암호화, 사용자 정의 SSL 인증서, AWS Certificate Manager (ACM) 지원, 웹 방화벽 통합 등 다양한 보안 기능을 제공한다. 또한, AWS Identity and Access Management (IAM) 및 AWS Shield와 같은 기능을 통해 권한 및 액세스 제어를 관리할 수 있다.
  • 요금제는 사용한 만큼 지불하는 Pay-As-You-Go 방식이 적용되며, 데이터 전송량과 요청 수에 따라 비용이 달라진다.

AWS Secrets Manager

  • AWS Secrets Manager는 애플리케이션, 서비스 및 IT 리소스에서 사용하는 비밀 정보를 안전하게 관리하고 액세스하는 서비스이다.
  • 데이터베이스 자격증명, API 키, OAuth 토큰 등과 같은 중요한 정보를 안전하게 저장, 회전, 검색 및 관리할 수 있다.
  • 자동 비밀 정보 회전 기능을 사용하면, 정기적으로 비밀 정보를 교체하여 보안을 강화할 수 있다.
  • 비밀 정보에 대한 액세스를 제어하기 위해 AWS Identity and Access Management (IAM) 정책을 사용하여 권한을 설정할 수 있다.
  • AWS Secrets Manager는 사용한 만큼 지불하는 Pay-As-You-Go 방식이 적용되며, 저장된 비밀 정보의 수와 API 호출 횟수에 따라 비용이 달라진다.

AWS Systems Manager Parameter Store

  • AWS Systems Manager Parameter Store는 애플리케이션 데이터와 시스템 설정을 안전하게 저장하고 관리할 수 있는 중앙화된 저장소 서비스이다.
  • 애플리케이션 구성 데이터, 암호화된 문자열, 데이터베이스 문자열 등과 같은 정보를 키-값 쌍으로 저장할 수 있으며, 이러한 파라미터를 사용하여 AWS 리소스와 서비스를 구성할 수 있다.
  • 파라미터를 계층적으로 구성할 수 있어, 리소스 및 애플리케이션 별로 쉽게 관리할 수 있다.
  • 액세스 제어를 위해 AWS Identity and Access Management (IAM) 정책을 사용하여 권한을 설정할 수 있다.
  • AWS Systems Manager Parameter Store는 대부분의 기능이 무료로 제공되지만, 고급 파라미터 기능을 사용할 경우 비용이 발생할 수 있다.
