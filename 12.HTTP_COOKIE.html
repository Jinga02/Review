Custom user model

  • django는 기본적으로 User Model을 제공한다.
  • 하지만 기본적으로 제공되는 필드들만으로는 원하는 서비스를 만들기 어렵다.
  • 그렇기 때문에 Custom User model을 사용해야 한다.
  • AUTH_USER_MODEL 참고 han-py
  • 프로젝트 중간에 AUTH_USER_MODEL 변경 매우 어려움

코드 작성하기

  1. accounts => models에 모델 정의
  2. settings에 AUTH_USER_MODEL = 'accounts.User' 정의
  3. admin.py에 User모델 등록

HTTP

  • HTTP 정의

    • HTML 문서와 같은 리소스들을 가져올 수 있도록 해주는 프로토콜(규칙, 규약)
    • 웹(WWW)에서 이루어지는 모든 데이터 교환의 기초
    • 클라이언트 - 서버 프로토콜이라고도 부름

  • HTTP 특징

    • 비연결지향 : 서버는 요청에 대한 응답을 보낸 후 연결을 끊는다.
    • 무상태 : 연결을 끊는 순간 통신이 끝나며 상태 정보가 유지되지 않는다.

쿠키(COOKIE)

  • 정의

    • 서버가 사용자의 웹 브라우저에 전송하는 작은 데이터 조각.
    • 웹 페이지 방문 시 방문 기록 등 브라우저에서의 정보들이 저장된 텍스트 파일.

  • 사용 목적

    • 세션 관리 : 로그인, 팝업체크, 장바구니 등의 정보 관리.
    • 개인화 : 사용자가 설정한 테마 등의 설정.
    • 트래킹 : 사용자 행동을 기록하고 분석.

  • 세션(session)

    • 일정 시간동안 같은 사용자(브라우저)로부터 들어오는 일련의 요구를 하나의 상태로 보고, 그 상태를 유지시키는 것.
    • 클라이언트가 서버에 접속하면 서버가 특정 session id를 발급하고, 클라이언트는 session id를 쿠키에 저장한다.

  • Lifitime(수명)

    • Session cookie : 현재 세션이 종료되면 삭제, 브라우저 종료와 함께 세션이 삭제된다.
    • Persistent cookies : Expires 속성에 지정된 날짜 혹은 Max-Age속성에 지정된 기간이 지나면 삭제된다.
