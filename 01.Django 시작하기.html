Framework란?

Framework의 사전적 의미
Frame(뼈대, 틀) + Work(일하다) 이다. 일정한 뼈대, 틀을 가지고 일을 하다.
즉 서비스 개발에 필요한 기능들을 미리 구현해서 모아 놓은것을 뜻한다.
따라서, Framework를 잘 사용하기만 하면 웹 서비스 개발에 있어서 필요한 모든 것들을 직접 개발할 필요없이,
내가 만들고자 하는 부분에만 시간을 투자하고 집중해 개발할 수 있다.

Framework의 장점과 단점

장점
  1. 소프트웨어의 생산성과 품질을 높여준다.
  2. 확장성이 좋으며 높은 완성도를 제공한다.
  3. 체계적인 코드 관리로 안정적인 유지보수가 가능하다.
단점
  1. 자유롭고 유연한 개발에 한계를 가진다.
  2. 코드 습득을 위한 학습시간이 필요하다.
  3. 프로젝트 용량이 증가할 수 있다.

Framework와 Library의 차이

Library란 소프트웨어를 개발할 때 컴퓨터 프로그램이 사용하는 비휘발성 자원의 모임. 즉 특정 기능을 모와둔 코드, 함수들의 집합이며 코드 작성 시 활용 가능한 도구들을 의미한다.
이러한 Library와 Framework의 차이는 아래와 같다.

프레임워크와 라이브러리의 차이점은 "제어 흐름"의 권한이 어디에 있는가이다.
라이브러리를 사용할 때 사용자는 애플리케이션 코드의 흐름을 직접 제어해야 합니다.
개발 시 필요한 기능이 있을 경우 능동적으로 라이브러리를 호출하여 사용하거나 기존에 구성된 함수나 코드를 가져다 써야 한다.
반면 프레임워크는 애플리케이션의 코드가 프레임워크에 의해 사용된다.
애플리케이션 코드는 프레임워크가 짜 놓은 틀에서 수동적으로 동작하기 때문에 제어의 흐름은 프레임워크가 가지고 있고 사용자가 그 안에 필요한 코드를 작성하게 된다.
여기서 제어의 역전(IoC, Inversion of Control)이란, 어떠한 일을 하도록 만들어진 Framework에 Control 권한을 위임하는 것을 의미하는데, 간단히 말해 프로그램의 제어 흐름 구조가 뒤바뀐 것을 뜻한다.
라이브러리의 경우 애플리케이션의 흐름을 사용자가 직접 제어해야 하지만 프레임워크의 경우 코드를 연결할 수 있는 위치를 제공하고 필요에 따라 사용자가 연결한 코드를 호출하는 제어 흐름 권한을 가지고 있다.

클라이언트-서버

  • 클라이언트란?
    • 서비스를 요청하는 주체
    • 웹 사용자의 인터넷에 연결된 장치(컴퓨터, 모바일...)
    • Chrome, Safari와 같은 웹 브라우저
  • 서버란?
    • 요청에 대해 서비스를 응답하는 주체
    • 웹 페이지, 사이트 또는 앱을 저장하는 컴퓨터
    • 클라이언트가 웹 페이지에 접근하려고 할 때 서버에서 클라이언트 컴퓨터로 웹 페이지 데이터를 응답해 사용자의 웹 브라우저에 표시됨

상호작용 예시

    예시 : Google 홈페이지에 접속
  1. 인터넷에 연결된 전세계 어딘가에 있는 구글 컴퓨터에게 'Google 홈페이지.html'파일을 달라고 요청
  2. 구글 컴퓨터가 나의 요청을 받고 'Google 홈페이지.html'파일을 인터넷을 통해서 나의 컴퓨터에게 응답
  3. 전달받은 'Google 홈페이지.html' 파일을 웹 브라우저가 나에게 볼 수 있도록 해석

위의 예시에서 'Google 홈페이지.html'을 요청한 컴퓨터를 클라이어트
'Google 홈페이지.html'를 제공한 컴퓨터,프로그램을 서버라고 한다.
즉 어떠한 자원(resource)를 요청(request)하는 쪽을 클라이언트, 자원을 제공해주는 쪽을 서버라고 한다.

Django를 배우는 이유

  1. Python으로 작성된 프레임워크
    • Python으로 만들어진 무료 오픈소스 웹 애플리케이션 프레임워크(web application framework)이다.
  2. 내부의 파이썬
    • Django는 'Zen of Python이라는 원칙을 따르고 있으며, 외부 라이브러리에 대한 뛰어난 지원과 프로그래머의 생산성을 향상시키는 것과 같은 파이썬의 모든 장점을 계승하고 있다.
  3. 검증된 Web Framework
    • 화해, Toss, 두나무, 당근마켓, 요기요 등 유명한 많은 서비스들이 사용하고 있다.
    • 즉, 안정적으로 서비스를 할 수 있다는점이 검증되어 있다.
