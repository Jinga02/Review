Admin Site

  • 일반사용자가 아닌 서버의 관리자가 사용하기 위한 페이지.
  • 모델 class를 등록하고 관리.
  • 간단한 구현
    1. python manage.py createsuperuser
      Username, Emaill address, Password 설정 # 관리자계정 생성
    2. server 실행 => 주소는 admin
    3. 모델 record를 보기 위해
      admin.py에 등록 # admin.site.register(class이름)
      4. from .models import class이름
    4. 다시 server 실행 => admin에 등록한 모델 생성됨(이전에 만들었던 class gui로 생성 및 관리 가능)

    • admin 화면

CREATE

create 로직을 구현

  • 사용자의 입력을 받을 페이지와 함수 form
  • 사용자가 입력한 데이터를 전송받아 DB에 저장하는 함수 create
  • Redirect()란
    • re(다시) + direct(지시하다)
    • 브라우저가 www.test.com/page1 URL을 웹 서버에 요청한다. 서버는 HTTP 응답 메시지를 통해 "www.test.com/page2로 다시 요청하세요~" 라고 브라우저에게 다른 URL(길, 방향)을 지시하는 것을 리다이렉트라고 한다.
    • render과 redirect의 차이 참고 ssungkang

  • new와 create view 함수 합치기

    1. request.method 값을 기준으로 if문 작성

    2. 불필요해진 new의 view 함수와 url path 삭제
    3. new.html에서 create.html로 이름 변경 및 action 속성 값 수정
    4. index 페이지에 있던 new관련 링크 create로 수정
>

DELETE

deltet()를 사용해 원하는 DB파일 삭제

  • 삭제하고자 하는 DB파일을 조회 후 삭제해야 한다.
  • 코드 작성 후 삭제버튼을 만들 html에 delete전송을 위해 form 작성

UPDATE

update()를 사용해 원하는 DB파일 수정

  • 수정도 CREATE 로직처럼 2개의 view함수가 필요하다.
  • 사용자의 입력을 받을 페이지를 렌더링 하는 함수 edit
  • 사용자가 입력한 데이터를 전송 받아 DB에 저장하는 함수 update

  • edit와 update view 함수 합치기

    1. request.method 값을 기준으로 if문 작성

    2. 불필요해진 edit의 view 함수와 url path 삭제
    3. edit.html에서 update.html로 이름 변경 및 action 속성 값 수정
    4. detail 페이지에 있던 edit관련 링크 update로 수정
>

공부하며 배운 명령어

django-admin startproject crud . # 현재 위치의 폴더에 바로 생성
auto_now_add=True # 생성될때 자동 부여
auto_now=True # 내용이 변경될때마다 자동 부여
from app이름.models import class이름
Articles.objects.all() # 전부 보기
Articles.objects.get(id=2) # Articles클래스 중 id가 2인것 가져와 1개만 보기
Article.objects.create(title = '게시글 4', content= '내용 4') # save를 하지 않아도 생성
views.py에 articles = Articles.objects.all() 간편하게 사용
