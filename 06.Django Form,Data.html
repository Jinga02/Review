HTML form element

  • 데이터가 전송되는 방법을 정의
  • 웹에서 사용자 정보를 입력하는 여러방식(text, button, submit등)을 제공하고, 사용자로부터 할당된 데이터를 서버로 전송하는 역할을 담당한다.
  • 핵심속성으로는 "action", "method"가 있다.

  1. action

    • 입력 데이터가 전송될 URL을 지정.
    • action을 지정하지 않으면 현재 form이 있는 페이지의 URL로 데이터를 보낸다.

  2. method

    • 데이터를 어떻게 보낼 것인지 정의.
    • GET방식과 POST방식 두가지 방법으로만 전송할 수 있다.
    • 참고 https://hyoje420.tistory.com/24

  3. input element

    • 사용자로부터 데이터를 입력 받기 위해 사용.
    • 'type' 속성에 따라 동작 방식이 달라진다.
      참고 http://jun.hansung.ac.kr/CWP/htmls/HTML%20Input%20Types.html
    • name
      • form을 통해 데이터를 제출(submit)했을 떄 name속성에 설정된 값을 서버로 전송하고, 서버는 name속성에 설정된 값을 통해 사용자가 입력한 데이터 값에 접근할 수 있다.
      • 서버에 전달하는 파라미터(name은 key, value는 value)로 매핑하는것이다.

  4. Quert String Parameters

    • 사용자가 입력 데이터를 전달하는 방법 중 하나로, url주소에 데이터를 파라미터를 통해 넘기는 것
    • Quert String라고도 불리며, &로 연결된 key=value 쌍으로 구성되며 기본 URL과 ?로 구분된다.

  5. form input으로 전송한 데이터 받아오기

    • 받아올 html파일 작성
    • 보낼 html파일에 action작성
    • views 파일 작성
    • 결과는 직접 작성한 후 확인

  6. 요청과 응답 객체 흐름

    1. 페이지가 요청되면 django는 요청에 대한 메타데이터를 포함하는 HttpRequest object를 생성한다.
    2. 해당하는 적절한 view 함수를 로드하고 HttpRequest를 첫번째 인자로 전달한다.
    3. 마지막으로 view함수는 HttpResponse object를 반환한다.
