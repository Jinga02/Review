Fixtures

  • 모델에 초기 데이터를 제공하는 방법
  • 필요성
    • 협업을 할 때 gitignore설정으로 인해 DB는 업로드 하지 않는다.
    이때 초기 데이터가 있어야 테스팅이 가능한 경우가 있는데,
    이러한 경우에 발생한 에러를 해결하기 위해 초기에 데이터를 제공해야한다.
  • migration와 fixtures를 사용해 data와 구조를 공유할 수 있다.

dumpdata 생성 (데이터 추출)

  1. 여러 모델을 하나의 json파일로 만들 수 있다.
    python manage.py dumpdata [app_name[.ModelName] app_name[.ModelName] ...] > {filename}.json
    데이터 꺼낼거야 근데 > 를 기준으로 이전의 데이터들을 > 이후에다가 뽑아줘
    예시) $ python manage.py dumpdata articles.article > articles.json
    폴더에 articles의 article데이터를 가진 articles.json파일이 생성됨.
    'Prettify JSON' 설치시 보기 좋게 json 출력가능 or --indent 4 추가 해주면 된다.
  2. 모든 모델을 한번에 dump하기 python manage.py dumpdata --indent4 > data.json
  3. 생성한 json파일은 폴더를 만들어 한곳에 모아주고, 원하는 app에 넣어줘서 기본주소를 설정해줘야 한다.

loaddata (데이터 입력)

  1. 데이터 로드해오기
    python manage.py loaddata articles.json user.json comment.json
  2. loaddata를 한번에 실행하지 않고 하나씩 실행한다면 모델 관계에 따라 순서가 중요하다.
    모델간의 상관관계를 따진 순서대로 load하거나 한번에 load를 해야 에러가 발생하지 않는다.
