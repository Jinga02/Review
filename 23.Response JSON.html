Response JSON

  • 지금까지와는 다르게 페이지(html)를 응답하는게 아닌, JSON데이터를 응답하는 서버로 변환 할 예정
  • JSON데이터를 받아 화면을 구성하여 사용자에게 보여주는 것은 Front-end Framework가 담당 할 예정
  • Front-end Framework는 Vue.js를 사용 할 예정
  • JSON을 응답하는 Django 서버 구성을 목표로 진행 할 예정

구현

사전준비

  1. 사용할 json파일과 함께 project(app, settings, urls...)준비
  2. fixtures파일을 load하여 초기 데이터 입력
    python manage.py loaddata articles.json

JsonResponse()를 사용한 JSON응답

html 하나를 응답하는 것이 아닌 JSON데이터를 응답해보기

  1. viesw 함수 작성
  2. 결과 확인

serializers를 사용한 JSON응답

JSON의 모든 필드를 모두 작성할 필요가 없어진다.

  1. views 함수 작성
  2. 결과는 JsonResponse와 동일

REST framework를 사용한 JSON응답

  • Restful API서버를 쉽게 구축 할 수 있도록 도와주는 오픈소스 라이브러리
  • Django의 Form, ModelForm클래스와 매우 유수하게 작동한다.
  1. pip로 djangorestframework 설치 후 settings => INSTALLED_APPS에 rest_framework입력
  2. 사용 할 app에 serializers.py 생성
  3. views 함수 작성
  4. 결과 확인 , JSON데이터를 DRF전용 템플릿으로 응답해준다.

Django REST framework - Single Model

단일 모델의 data를 serialization하여 JSON으로 변환하는 방법

  • 사전 준비(실습 시 위에서 만들 프로젝트를 사용)
    1. postman설치

      2. postman이란 API를 구축하고 사용하기 위한 플랫폼이다.
      API를 빠르게 만들 수 있는 여러 도구 및 기능을 제공한다.

    2. 사용 할 프로젝트 생성, 앱 생성 및 초기 세팅
    3. 생성 한 앱에 모델 작성 및 migration 진행
    4. fixtures 데이터 load

      5. python manege.py loaddata 생성 한 앱.json

    5. DRF설치, 등록 및 패키지 목록 업데이트

      6. pip install djangorestframework

  • 모든 게시글 조회
    1. 사용 할 urls와 serializer, views작성
    2. postman에서 결과확인
  • 게시글 상세보기
    1. urls 작성
    2. views 작성
    3. postman에서 결과확인
  • 게시글 작성하기
    1. 모든 게시글 조회 views 함수 수정
    2. postman POST, body에 양식 맞춰 게시글 작성
  • 게시글 수정하기
    1. 게시글 상세보기 views 함수 수정
    2. postman PUT, body에 양식 맞춰 게시글 작성
  • 게시글 삭제하기
    1. 게시글 상세보기 views 함수 수정
    2. postman DELETE, body에 양식 맞춰 게시글 작성
